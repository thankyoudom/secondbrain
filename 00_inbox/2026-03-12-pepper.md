# Pepper

Great question — and good timing to think through this before tomorrow. Here’s the honest picture:
Can you get something going in an hour? Yes, but with a narrow scope.
The beginner-friendly path is Choregraphe, a visual block-based programming tool where you connect action blocks for movement, speech, and speech recognition — no deep coding required. ? A simple “greet the patient, say a scripted message, display info on the tablet” flow is genuinely achievable within an hour if your environment is set up.
### What “an hour” realistically gets you
|  | ? | Pepper greeting visitors by name / role (“Welcome to the clinic!”) |
| --- | --- | --- |
|  | ? | A scripted spoken intro with arm gestures |
|  | ? | Tablet displaying a menu or info screen |
|  | ? | Basic touch-triggered responses |
The real friction points to prepare for:
|  | 1. | Setup time eats into your hour. You need Choregraphe installed, the robot on your network, and the Choregraphe version matched to Pepper’s NAOqi OS version. ? Do this tonight, not tomorrow. |
| --- | --- | --- |
|  | 2. | Choregraphe is old and quirky. It works, but the interface feels dated and the documentation assumes NAOqi 2.x. Budget time for small surprises. |
|  | 3. | SoftBank stopped manufacturing Pepper in 2021, so official support is gone ? — community resources and GitHub repos are your best bet if you hit walls. |
|  | 4. | The overall software architecture can be frustrating at first — it typically takes a day or two to really get comfortable with how the modules combine. ? |
### My suggestion for a clinic demo
Keep it extremely tight in scope. One linear flow:
|  | ? | Pepper detects someone approaching (or you trigger it manually) |
| --- | --- | --- |
|  | ? | Speaks a welcome message relevant to the clinic |
|  | ? | Displays something on the tablet (wait time, a map, a logo) |
|  | ? | Maybe one gesture/animation |
Don’t try to build dialog trees or voice recognition for tomorrow — that’s where things get complicated fast. A confident, polished simple demo lands better than an ambitious one that glitches.
