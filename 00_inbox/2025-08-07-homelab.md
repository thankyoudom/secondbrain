? Step 1: Set up Prometheus + Grafana on Server A
Install Docker + Docker Compose
Use this docker-compose.yml as a starting point:
version: '3'

### services
### prometheus
image: prom/prometheus
### volumes
- ./prometheus.yml:/etc/prometheus/prometheus.yml
### ports
- "9090:9090"

### grafana
image: grafana/grafana
### ports
- "3000:3000"
### volumes
- grafana-storage:/var/lib/grafana

### volumes
### grafana-storage
? Step 2: Install Node Exporter on all 4 laptops
This exposes metrics to Prometheus.
### For Linux
docker run -d -p 9100:9100 --name node-exporter prom/node-exporter
### For macOS & Windows
Use the official Node Exporter binaries, or build manually.
Make sure port 9100 is accessible.
? Step 3: Configure Prometheus to scrape all machines
Update your prometheus.yml config to look like:
### global
scrape_interval: 15s

### scrape_configs
- job_name: 'laptop-monitoring'
### static_configs
- targets:
- 'localhost:9100'      # Server A
- '192.168.1.101:9100'  # Server B
- '192.168.1.102:9100'  # Mac
- '192.168.1.103:9100'  # Windows
Replace IPs with actual LAN IPs of each device.
? Step 4: Start Grafana and build dashboards
Access Grafana on [<Server](http://<Server) A IP>:3000
Add Prometheus as a data source
Import dashboards from Grafana Labs (e.g., Node Exporter Full)

? Bonus: Add Logging with Loki
Loki collects logs from the devices.
You can use Promtail or Fluentbit to ship logs from Mac, Windows, Linux into Loki.
Visualize them in Grafana.

? Optional: File Movement Between Devices
This isn’t part of Prometheus/Grafana, but you can add:
? Option 1: rsync or scp (manual or automated)
Works between Linux/Mac easily
Can use cron or Python to schedule jobs
? Option 2: Syncthing
Runs on all OSes
GUI-based sync across devices
Secure, encrypted, LAN-friendly
You can monitor Syncthing health via Prometheus, too, using community exporters.
